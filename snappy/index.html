Snappy

Web App

Retro camera effects are hip. HTML5 is hip. So we combined them.

Apps like Instagram have repopularized classic camera effects, but these apps are typically limited to closed platforms like iOS and Android. So when the Mozilla Apps Hack Day 2012 challenged us to build open, cross-platforms web apps, a couple other Mozilla interns and I decided to build an Snappy, Instagram-inspired app. My teammates handled the graphics editing algorithms while I worked on the design and front-end of the app. We ended up completing the app in five hours and winning first place.

Design

Off the bat, I wanted to use arrange the photo-editing process as a few distinct steps to convey its simplicity. The layout went through many iterations, finally arriving at a main content area with a sidebar displaying the steps. The sidebar's color changes with each step to further demonstrate progress.

Fitting with the nature of a hackathon, I had to work around various technical limitations as they came up. For example, though we initially wanted to show each effect's result with a thumbnail preview, that feature ended up being infeasible, performance-wise, so I used a text-based list of tabs attached to the photo instead.

Technology

Snappy is a simple, client-side JavaScript app. I contributed to the code for front-end interactions, setting up events to trigger when the buttons were pressed or when the app moved to the next step.

During the hackathon, we didn't concentrate on code style much, so our code became one huge mess. After the fact, I organized and modularized it a bit. The code is on GitHub.
